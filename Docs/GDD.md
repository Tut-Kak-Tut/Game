## 1. Vision & Pillars

Гра — повільне, насичене лором подорожування великим фентезі-світом, поєднане з action-RPG бойовкою і глибокою системою білдів. Гравець сам обирає темп: можна йти за головною сюжетною лінією, а можна загубитися в дослідженні.

**Чотири стовпи дизайну:**

1. **Світ винагороджує цікавих.** Будь-який рух у бік від стежки веде до записки, руїни, унікального предмета або прихованого квесту. Жодна локація не існує «просто для краси».
2. **Сила приходить з білду, а не з рівня.** Стартовий клас + дерево пасивів + легендарні предмети визначають персонажа значно сильніше, ніж сирий ровень.
3. **Смерть має значення — за вибором гравця.** Standard-режим прощає, Hardcore — ні. Гравець свідомо обирає рівень напруги при створенні світу.
4. **Світ живий і повільний.** Темп Tibia/Albion (NPC, торгівля, спостереження) поєднаний з action-бойовкою Diablo/PoE.

## 2. Target Audience & Inspirations

**Цільова аудиторія:** гравці, що цінують MMO-світи (Tibia, Albion Online) і ARPG-білди (Diablo II, Path of Exile, Last Epoch). Терплячі до повільнішого темпу, мотивовані дослідженням і кастомізацією.

**Орієнтири за відчуттям:**
- **Tibia / Albion** — структура живого світу, вага торгівлі, повільне зростання.
- **Diablo II / Path of Exile** — action-бойовка, рідкісні легендарні предмети, простір для білдів.

**Не є орієнтиром:** Souls-like складність бою, рогалики з повним рестартом (крім опційного Hardcore), JRPG-наративи з пошагровим боєм.

---

## 3. Setting, Tone & Lore Direction

**Сетинг:** класичне фентезі. Королівства людей сусідують з ельфійськими лісами та гномячими горами; між ними — давні руїни, забуті храми, поховання богів старшого покоління. Магія — невід'ємна частина світу, але її розуміють і контролюють лише обрані.

**Тон:** пригодницький, з нотками таємниці. Світ переважно «ясний» (не grimdark), але має темні куточки — прокляті ліси, занедбані шахти, культи.

**Лор-напрями (TBD):**
- Імена королівств, рас, божеств.
- Ключова катастрофа в історії, що сформувала нинішній світ.
- Конкретні фракції та їхні цілі.

Лор розкривається **не через експозицію**, а через знахідки в світі: записки, таблички, описи легендарних предметів, монологи NPC.

## 5. Core Gameplay Loop

```
   ┌─────────────────────┐
   │ 1. Дослідження зони │
   │ Знайдено записку /  │
   │ табличку / руїну    │
   └──────────┬──────────┘
              │ → quest hook
   ┌──────────▼──────────┐
   │ 2. Підготовка у хабі│
   │ Закупівля, апгрейд  │
   │ пасивів, ремонт     │
   └──────────┬──────────┘
              │
   ┌──────────▼──────────┐
   │ 3. Похід у локацію  │
   │ Бій → лут → ризик   │
   │ смерті              │
   └──────────┬──────────┘
              │
   ┌──────────▼──────────┐
   │ 4. Повернення в хаб │
   │ Продаж, ідентифік., │
   │ прокачка білду      │
   └──────────┬──────────┘
              │ → нова зачіпка
              ▼ (повторити)
```

**Підпетля бою:** WASD рух → sprint/dash → автоатака зброєю + скіли з бару → реактивний кітинг / танкування → лут після бою.

## 7. Combat & Death

### Бойовка

Action-style із миттєвими хітами, кулдаунами, костами мани/стаміни. Підтримується через [`SpellCasterRuntime`](Systems-Effects.md) та [`CharacterStats`](Systems-Character.md).

- Типи дамагу: **Physical / Magic / True** (✅ `DamageType` в `GlobalEnums.cs`).
- Міткація: armor для Physical, magic resistance для Magic, True ігнорує захист (✅ `CharacterStats.TakeDamage`).
- ⬜ **Player melee attack** — наразі гравець не має автоатаки ближнього бою. Лише скіли. Потрібно додати `PlayerAttackController`.
- ⬜ **Ranged спели з проджектайлами** — `SpellEffects` уже підтримує миттєві хіти, але снаряди з траєкторією треба додати.
- ⬜ **Knockback / stagger** — поки відсутні; додати як механіку для контролю натовпу.

### Death Modes (вибір при створенні світу)

Гравець обирає режим **один раз при створенні слоту** — пізніше змінити не можна.

| Режим | Поведінка при смерті |
|---|---|
| **Standard** ⬜ | Респ у хабі. Втрата ~10% поточного XP до наступного рівня + ~10% золота. Спорядження залишається. Quest-progress зберігається. |
| **Hardcore** ⬜ | Перманентний кінець. Сейв переноситься в архів `saves/graveyard/`, на головному меню видно як «полеглого героя». Подальше завантаження неможливе. |

Hardcore — **строго PvE** (див. секцію 22).

## 8. Character Progression

### Атрибути

✅ Шість основних атрибутів через [`CharacterStats`](Systems-Character.md):

| Атрибут | Скорочення | Впливає на |
|---|---|---|
| Strength | Str | Physical damage, max stamina |
| Agility | Agi | Crit chance, рух, dash |
| Constitution | Con | Max health, регенерація |
| Intelligence | Int | Spell power, max mana |
| Wisdom | Wis | Mana regen, magic resistance |
| Charisma | Cha | Ціни у торгівців, опції в діалогах |

### Рівень та XP

✅ XP-система через `RuntimeEventBus` (`XpGrantedEvent`, `LevelUpEvent`). Кожен рівень → атрибут-поінти + поінти пасивного дерева.

### Класи (стартовий вибір)

⬜ Потрібно: створити `ClassDefinition` ScriptableObject з полями: `subclassId`, стартові статти, стартові скіли (`SpellDefinition[]`), стартова точка в `SkillTreeDefinition`, стартовий інвентар, стартовий компаньйон (опційно).

**Три базові класи з підкласами** (детальний розпис у секції 17):

- **Чаклун** — Стихійний / Містичний / Темний / Світлий
- **Воїн** — Паладин / Асасин / Монах / Варвар
- **Слідопит** — Лучник / Снайпер / Мисливець

### Дерево пасивів

✅ `SkillNodeDefinition` + `SkillTreeDefinition` уже підтримують:
- Передумови (Level, ParentNode, QuestComplete, TagOwned)
- Пасивні стат-бонуси
- Respec-режими (FullReset, BranchReset, SafeReset)
- Збереження через SaveId

Дерево спільне для всіх класів; підклас задає лише точку входу. Це гібрид Diablo II (стартова позиція) + PoE (велике дерево, гнучкість білдів).

## 9. Loot Philosophy (Hybrid)

Поєднує **потік Diablo/PoE** з **вагою Tibia**.

| Тир | Назва | Дроп | Призначення |
|---|---|---|---|
| **T1** | Junk / Currency | Постійний | Золото, срібло, краф-матеріали, расходники. Основа економіки. |
| **T2** | Magic / Rare | Регулярний | Випадкові афікси (PoE-стиль). Основа потоку лута, drives grind. |
| **T3** | Uniques / Legendaries | Дуже рідкісний | Іменовані предмети з фіксованими статтями і власним лором. Tibia-стиль — кожна штука «впізнавана». |

**Реалізація** (⬜ потрібно):
- `LootTable` ScriptableObject — список можливих дропів з вагами.
- `AffixDefinition` ScriptableObject — пул префіксів/суфіксів для T2.
- `UniqueItemData : ItemData` — для T3 (фіксовані статти + опис лору).
- Драйвери дропу: бос, скриня, секретна локація, артефактник (через craft).

## 10. Story, Quests & Discovery

### Головний квест

⬜ Опційна лінійна сюжетна арка від стартової локації до фінального боса. **Не примусовий** — гравець може ігнорувати головний квест нескінченно.

### Emergent Quests (основа гри)

⬜ Більшість квестів видається **через знахідки в світі**:
- **Записки** — короткі тексти від NPC або давніх героїв.
- **Кам'яні таблички** — лор + іноді квест-зачіпка.
- **Руїни** — дослідження видає завдання («знайти решту фрагментів»).
- **Випадкові події** — NPC у небезпеці на дорозі, караван під атакою.

### Технічна реалізація

⬜ Потрібно створити:
- `QuestData` ScriptableObject — кроки, нагороди, передумови.
- `LoreItem : ItemData` — записки/таблички, що видають quest hook при підборі.
- `QuestService` (новий компонент під `GameSession`) — стан квестів, активні цілі.
- `RuntimeEventBus` події: `QuestStartedEvent`, `QuestStepCompletedEvent`, `QuestCompletedEvent`.

### Quest Journal

⬜ UI журналу: активні квести, виконані квести, знайдений лор (як окрема вкладка). Hotkey J.

## 11. NPCs, Dialogue, Vendors

### NPC

⬜ Базовий `NpcController`:
- Спрайт + анімація idle/talk
- Interact-радіус (E для взаємодії)
- Прив'язка до `DialogueTree` ScriptableObject
- Опційно: розклад (день/ніч присутність — див. секцію 21)

### Діалоги

⬜ `DialogueNode` ScriptableObject: репліка + варіанти відповідей + умови (квести, статти, Charisma) + наслідки (видати квест, відкрити торгівлю).

### Торгівці

⬜ `VendorComponent` на NPC:
- Інвентар торгівця (фіксований або згенерований з `LootTable`)
- Купівля/продаж зі знижкою на основі Charisma
- Інтеграція з `Inventory` (✅ частково готово) і `Wallet` (⬜).

## 12. World Map & Travel

⬜ Дві UI-системи:

1. **World Map** — мапа всіх 5 регіонів, відкривається на M. Показує відомі локації, активні квести, fast-travel waypoints.
2. **Minimap** — компактна карта поточної зони у куті екрана. Показує найближчих ворогів, інтерактивні точки.

**Fast travel:** через відкриті waypoints у хабі та крупних локаціях. Перший прохід — пішки/маунтом, далі — телепорт.

## 13. Save System & Modes

### Поточний стан

✅ `SaveService` + `ISaveDataProvider` API в `Game.CoreRuntime`.
🟡 `SpellCasterRuntime` і `SkillProgressionService` уже реалізують провайдери.
⬜ **Disk I/O відсутній** — наразі сейв тримається в пам'яті.

### Що потрібно додати

⬜ **Disk persistence:** JSON у `Application.persistentDataPath/saves/`.
⬜ **Slot-based saves** з мета-даними:
```json
{
  "slotId": "...",
  "playerName": "...",
  "className": "Чаклун (Стихійний)",
  "level": 12,
  "mode": "Standard" | "Hardcore",
  "playtimeSeconds": 4521,
  "lastPlayed": "2026-05-10T18:23:00Z",
  "regionId": "ForestOfElders"
}
```
⬜ **Hardcore-логіка:**
- При смерті — slot переноситься в `saves/graveyard/<slotId>/`.
- Завантаження архівованих сейвів заблоковано (тільки перегляд статистики).
- На головному меню — окрема вкладка «Полеглі герої».

⬜ **Зональний state** — стан кожного регіону (вбиті боси, відкриті скрині, виконані квести) серіалізується як окремий чанк через `ISaveDataProvider`.

## 15. Co-op Considerations (Future)

MVP — соло. Але рішення зараз впливають на можливість коопу пізніше.

**Що добре:**
- Стан гри живе в компонентах + `GameSession`-сервісах → шлях до server-authoritative state.
- `RuntimeEventBus` зі struct-events → готовий до перетворення в network events.
- Контент через ScriptableObject-асети → легко синхронізується між клієнтами (всі мають той самий `SpellDefinition`).

**Чого уникати вже зараз:**
- Глобальних `static` змінних поза сервісами.
- Прямого доступу до `Input` всередині `Update()` ігрових об'єктів. **Виносити input у окремий шар** (input → intent → action).
- Логіки, яка передбачає одного гравця (наприклад, hardcoded `playerTarget` у `EnemyAI`).

**Це ще не TODO** — лише дисципліна на майбутнє.

- PvP-арени (тільки Standard)

## 17. Classes & Subclasses (детально)

Три базові класи. Кожен має 3-4 підкласи — гравець обирає **одне поєднання базовий+підклас на старті**, змінити неможливо.

### 17.1 Чаклун (Mage)

Дальня магія, низьке здоров'я, висока маневреність через скіли. Основні атрибути: **Int + Wis**.

| Підклас | Тематика |
|---|---|
| **Стихійний** | Вогонь, лід, блискавка. Високий бурст-дамаг, AoE. |
| **Містичний** | Аркана, телепортація, контроль. Маневреність + дебафи. |
| **Темний** | Прокляття, нежить, life-drain. DoT, призов. Опційно — компаньйон-нежить. |
| **Світлий** | Лікування, баффи, контроль нежиті. Підтримка та анти-undead. |

### 17.2 Воїн (Warrior)

Ближній бій, висока витривалість, варіативність ролі. Основні атрибути: **Str + Con**.

| Підклас | Тематика |
|---|---|
| **Паладин** | Танк/підтримка з лікуванням і захисними аурами. |
| **Асасин** | Скритність, кинджали, висока критичність. (Назва за вибором користувача — «ассосін».) |
| **Монах** | Беззбройний бій, ухилення, контр-атаки, внутрішня енергія. |
| **Варвар** | Двуручна зброя, лють, життя-за-силу. |

### 17.3 Слідопит (Ranger)

Дальній бій, кайтінг, природа, сліди. Основні атрибути: **Agi + Wis**.

| Підклас | Тематика |
|---|---|
| **Лучник** | Класичний рейнджер з луком, AoE-стріли, баффи команди. |
| **Снайпер** | Точкові постріли з арбалета, висока крит-шкода, повільний темп. |
| **Мисливець** | Пастки, тваринні компаньйони (**до 3 одночасно**, див. секцію 19), сліди. |

### Реалізація

⬜ `ClassDefinition` ScriptableObject:
```csharp
public class ClassDefinition : ScriptableObject {
    public string baseClass;       // "Чаклун" | "Воїн" | "Слідопит"
    public string subclass;
    public string description;
    public Sprite icon;
    public AttributeBlock startingStats;
    public SpellDefinition[] startingSkills;
    public SkillNodeDefinition skillTreeEntryNode;
    public ItemData[] startingInventory;
    public CompanionDefinition defaultCompanion; // опційно
    public int maxCompanions; // 1 за замовчуванням, 3 для Мисливця
}
```

## 18. Crafting (4 професії)

Ремесла відкриваються через NPC-майстрів у хабі. Гравець може опанувати всі чотири, але прогресія в кожній окрема (свої рівні, свої рецепти).

| Професія | Що робить | Особливість |
|---|---|---|
| **Коваль** | Зброя ближнього бою, броня, заточка/ремонт | Покращення T2 предметів афіксами |
| **Ювелір** | Кільця, амулети, інкрустація каменями в гнізда | Універсальний для всіх класів |
| **Артефактник** | Магічна зброя, посохи, артефакти | **Шанс створити випадковий T3 unique** — джерело легендарок поза боса́ми |
| **Бібліотекар / Писар** | Свитки, скрипти заклинань, рецепти, мапи | Постачає одноразові розхідники |

⬜ Реалізація:
- `CraftingProfession` ScriptableObject — рівні, перелік рецептів.
- `Recipe` ScriptableObject — інгредієнти, час, рівень професії, результат, шанс на бонус (для артефактника — `uniqueChancePercent`).
- `CraftingService` — старт крафту, обчислення результату, інтеграція з `Inventory`.

## 19. Mounts & Companions

### Маунти

⬜ **Скакуни** — підвищують швидкість пересування поза боєм. Особливості:

- Виклик/злізти через hotkey R.
- Окремий стан персонажа (`PlayerController` має враховувати `IsMounted`).
- Бій з маунта — TBD (рекомендовано: автоматично злазити при взятті дамагу або початку бою).
- NavMesh-сумісність — маунти можуть мати інші обмеження прохідності (не лізуть у вузькі тунелі).
- Перші маунти даються через головний квест або як рідкісна нагорода.

### Компаньйони

⬜ **AI-союзник**, що б'ється поруч.

- **За замовчуванням — 1 компаньйон** на гравця (для більшості класів).
- **Окремі підкласи** — наприклад, **Мисливець** — можуть мати **до 3 одночасно** (`ClassDefinition.maxCompanions`).
- Реалізація: новий `CompanionAI` компонент. **Використовує** `CombatantBehaviour` і `CharacterStats` (так само як вороги). Розширює існуючий `EnemyAI` патерн, але з «дружнім» таргетингом.
- Компаньйон отримує накази (follow / stay / attack target) через UI або hotkey.
- Death-логіка: компаньйон може загинути; в Standard режимі — оживлюється у хабі; в Hardcore — TBD (може бути перманентна втрата).

## 20. Economy

### Валюти

⬜ Дві монети:

| Монета | Курс |
|---|---|
| **Золота** | 1 зол = **100 срібних** |
| **Срібна** | базова |

Дрібні витрати (зілля, ремонт, базові розхідники) у срібних. Спорядження, дороге крафт-сировину, легендарки — у золотих.

### Реалізація

⬜ `Wallet` компонент на гравцеві:
```csharp
public class Wallet {
    public int silver;
    public int gold;
    public int TotalInSilver => gold * 100 + silver;
    public bool TryDeduct(int silverCost);
    public void Add(int silver, int gold = 0);
}
```

Інтеграція:
- Vendor UI — показує ціни в зол/ср, конвертує автоматично.
- Drop tables — золото випадає у срібному еквіваленті, конвертується автоматично коли набирається 100.
- Crafting — рецепти можуть мати валютну вартість.
- Charisma впливає на **ціни купівлі** (нижчі) і **ціни продажу** (вищі) у торгівців.

## 21. Day/Night Cycle

⬜ **Є, але не центральна механіка.**

### Реалізація

- Глобальний `TimeOfDay` сервіс під `GameSession`. Один цикл = ~30 хв реального часу (TBD).
- `LightingController` — змінює освітлення сцени (URP global volume).
- Подія `TimeOfDayChangedEvent` через `RuntimeEventBus`.

### Геймплейний вплив

- **NPC-розклади** — деякі торговці тільки вдень, деякі тільки вночі.
- **Спавн ворогів** — нічна нежить, нічні хижаки.
- **Нічні квести** — особливі quest hooks тільки після заходу сонця.
- **Опційний спочинок** у тавернах для пропуску ночі.

### Чого уникати

- Примусового сну.
- Стелс-розділів, які працюють лише вночі.
- Систем, які ламаються без day/night (тобто весь основний геймплей має працювати незалежно від часу доби).

## 23. Regions & Streaming Architecture

### Масштаб

- **MVP — 5 регіонів.**
- **Архітектура має масштабуватися до 20+** регіонів без переробки.
- Це **критична архітектурна вимога** — закласти зараз, переробляти на 20 регіонах буде дорого.

### Сцени

⬜ Кожен регіон — окрема Unity-сцена:
```
Assets/Game/Scenes/
├── SampleScene.unity            (поточна — буде перепризначена або видалена)
├── Tutorial/
│   └── TutorialOrigin.unity
├── Hub/
│   └── CentralHub.unity
└── Regions/
    ├── Region_ForestOfElders.unity
    ├── Region_FrozenPeaks.unity
    ├── Region_BlackMarsh.unity
    ├── Region_SunBurntDesert.unity
    └── Region_StormCoast.unity
```

### Streaming

⬜ Новий `SceneStreamingService` під `GameSession`:
- Перехід через `SceneManager.LoadSceneAsync` (не блокує main thread).
- Завантаження тільки активного регіону + хабу + туторіалу (опційно).
- Вивантаження неактивних сцен для економії пам'яті.
- Zone state (вбиті боси, відкриті скрині, виконані квести) серіалізується через `ISaveDataProvider` як `RegionSaveChunk`.

### SceneRuntimeBootstrap — потрібно розширити

- ✅ Поточний `SceneRuntimeBootstrap` працює лише на `SampleScene`.
- ⬜ Має підтримати багатосценний контекст:
  - `GameSession` — `DontDestroyOnLoad` (зберігається між сценами).
  - `DamageTextManager`, `CharacterStats` рефреси — переробити на on-scene-load hook.
  - Player — переноситься між сценами або інстантіюється в кожному регіоні з його збереженого стану.
